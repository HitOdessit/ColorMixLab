# Development notes (AI-generated archive)

This directory contains 54 markdown files generated by Claude Code during the development of ColorMixLab. They document features, fixes, optimizations, and migrations as they happened — essentially a real-time engineering journal produced by the AI as it wrote the code.

These were originally scattered across the repo root before being archived here during cleanup. They are preserved as a transparent record of how an AI-assisted build actually unfolds: every animation tweak, every layout iteration, every cross-platform migration step.

## What's in here

The files fall into roughly these categories:

- **Animation/UI iteration** — `ANIMATION_*`, `CELEBRATION_*`, `DYNAMIC_BUBBLE_SIZING.md`, `MORPHING_ANIMATION_VISUAL_GUIDE.md`, `FLYING_BUBBLES_ANIMATION.md`
- **Math challenge feature work** — `MATH_CHALLENGE_*`
- **Layout fixes (landscape/tablet)** — `LANDSCAPE_*`, `TABLET_RESPONSIVE_DESIGN.md`, `THREE_COLUMN_INTRO_IMPLEMENTATION.md`
- **KMP migration** — `KMP_MIGRATION_*`, `MIGRATION_COMPLETE.md`
- **iOS port** — `IOS_*`, `XCODE_SETUP_GUIDE.md`
- **Bug fixes & optimizations** — `BUG_FIXES_AND_OPTIMIZATIONS.md`, `FREEZE_FIX_SUMMARY.md`, `ZERO_FREEZE_OPTIMIZATION.md`, `PERFORMANCE_OPTIMIZATION.md`
- **Build / project setup** — `BUILD_INSTRUCTIONS.md`, `REBUILD_INSTRUCTIONS.md`, `EMULATOR_LAUNCH_GUIDE.md`, `AUTO_VERSION_SETUP.md`, `VERSION_MANAGEMENT.md`

## Why keep them?

Two reasons:

1. **Honesty about the AI-build process.** A polished portfolio repo could quietly delete these. Keeping them visible shows the unfiltered shape of AI-assisted development — including the places where features needed multiple iterations.

2. **Useful artifacts.** Some are genuinely informative (e.g., `KMP_MIGRATION_GUIDE.md`) and could help someone understanding the codebase or attempting similar work.

## What's missing

These notes are output, not input. They document what was built; they don't expose the prompts that produced them. For example prompts you could use to reproduce parts of this project, see [`docs/reproduce.md`](../reproduce.md).
